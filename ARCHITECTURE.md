# Архитектура

## Обзор

Приложение состоит из React-клиента, реактивного Spring Boot API, PostgreSQL и локального файлового хранилища.

```text
React + TypeScript (5173)
          |
          | /api через Vite proxy
          v
Spring Boot WebFlux (8081)
          |
          +---- R2DBC ----> PostgreSQL (5433)
          |
          +--------------> data/uploads
```

## Backend

Backend использует Java 21, Spring Boot 4, Spring WebFlux и Spring Data R2DBC.

Код разделён по предметным областям:

```text
src/main/java/com/example/filestorage
├── bookmark   — закладки
├── folder     — папки
├── note       — заметки
├── shared     — общие ошибки и обработчик исключений
└── storage    — файлы и физическое хранилище
```

Каждая область содержит контроллер, сервис, модель, репозиторий и при необходимости DTO запроса.

### Реактивный поток

Контроллеры возвращают `Mono` или `Flux`. Репозитории работают через R2DBC без блокирующего JDBC. Создание сущностей с UUID выполняется через `R2dbcEntityTemplate.insert`, а обновление существующих сущностей — через `ReactiveCrudRepository.save`.

### API

| Область | Базовый адрес | Операции |
|---|---|---|
| Папки | `/api/folders` | список, создание, переименование, удаление |
| Файлы | `/api/files` | список, загрузка, скачивание, удаление |
| Закладки | `/api/bookmarks` | список, создание, изменение, удаление |
| Заметки | `/api/notes` | список, создание, изменение, удаление |

Для получения вложенных папок используется `GET /api/folders?parentId={id}`. Для получения файлов папки используется `GET /api/files?folderId={id}`.

## База данных

PostgreSQL запускается из `compose.yaml` и доступен приложению на порту `5433`.

Схема находится в `src/main/resources/schema.sql` и применяется при каждом запуске благодаря `spring.sql.init.mode=always`.

### Таблицы

| Таблица | Назначение |
|---|---|
| `folders` | иерархия папок через `parent_id` |
| `stored_files` | метаданные загруженных файлов |
| `bookmarks` | названия, адреса и описания ссылок |
| `notes` | заголовки, тексты, цвета и даты заметок |

Папки образуют дерево через внешний ключ `folders.parent_id -> folders.id`. Файлы связаны с папкой через `stored_files.folder_id`.

## Файловое хранилище

PostgreSQL хранит только метаданные файла. Содержимое записывается в `data/uploads`, а уникальное имя физического файла хранится в поле `storage_name`.

Поток загрузки:

```text
multipart-запрос
  -> FileController
  -> FileStorageService
  -> запись в data/uploads
  -> INSERT метаданных в stored_files
```

При ошибке сохранения метаданных сервис удаляет уже записанный физический файл. При удалении файла сначала проверяется запись, затем удаляются физический файл и метаданные.

## Frontend

Frontend находится в `frontend` и использует React 19, TypeScript и Vite.

Основная логика приложения расположена в `frontend/src/App.tsx`, стили — в `frontend/src/App.css`.

### Состояние

React хранит:

- активный раздел;
- текущий путь и содержимое папки;
- раскрытые ветки дерева;
- файлы, закладки и заметки;
- состояния форм, загрузки, уведомлений и подтверждений.

### Дерево папок

Содержимое папки загружается только при первом раскрытии ветки. Результат сохраняется в `expandedFolders`. Повторный клик удаляет ветку из состояния и скрывает дочерние элементы. Компонент рендерит дочерние папки рекурсивно.

Кнопка «Открыть» меняет текущий путь. Создание папки и загрузка файла выполняются относительно текущей открытой папки.

### Заметки

Поиск заметок выполняется на клиенте по заголовку и тексту. Создание и изменение используют одну форму. Допустимые цвета ограничены одинаковым набором на frontend и backend: `white`, `yellow`, `green`, `blue`, `rose`.

### Визуальный слой

- Framer Motion управляет переходами и анимациями;
- Lucide React предоставляет иконки;
- Vite перенаправляет `/api` на `http://localhost:8081`;
- интерфейс адаптируется на контрольных ширинах `980px` и `620px`.

| Frontend | `http://localhost:5173` |

| PostgreSQL | `localhost:5433` |
| Файлы | `data/uploads` |
| Backend-конфигурация | `src/main/resources/application.properties` |
| Vite-конфигурация | `frontend/vite.config.ts` |

## Проверка

Backend проверяется командой `./mvnw.cmd test`, frontend — командой `npm run build` из каталога `frontend`.
